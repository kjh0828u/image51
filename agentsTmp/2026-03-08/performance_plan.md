# BrushEditor 성능 개선 계획

`BrushEditor.tsx`에서 발생하는 랙(lag) 현상을 해결하기 위한 단계별 최적화 계획입니다.

## 1. 불필요한 React 리렌더링 제거
- **문제**: `handleMouseMove`에서 `setMousePos`를 호출하여 모든 마우스 이동마다 2000라인 이상의 대형 컴포넌트가 전체 리렌더링됨.
- **해결**: 
  - `mousePos` 상태를 제거하거나 별도의 리프 컴포넌트(StatusBar)로 분리.
  - 마우스 좌표 업데이트는 Ref를 사용하여 DOM 요소를 직접 조작하거나, 상태 업데이트 빈도를 제한(Throttling).

## 2. 선택 영역 외곽선 알고리즘(Marching Squares) 최적화
- **문제**: 고해상도 이미지(예: 4000px 이상)에서 `drawMarching`의 이중 루프가 메인 스레드를 점유하여 심각한 성능 저하 발생.
- **해결**:
  - **다운샘플링**: 외곽선 계산용 마스크를 저해상도(예: 최대 너비 1000px 정도)로 축소하여 계산하고, 그릴 때만 다시 확대.
  - **캐싱 강화**: 선택 영역이 바뀌지 않았을 때는 세그먼트 재계산을 철저히 방지.
  - **Worker 검토**: 계산량이 여전히 많다면 Web Worker로 분리. (현재는 다운샘플링만으로도 충분할 것으로 보임)

## 3. 루프 및 연산 최적화
- **문제**: `floodFillSelect`, `expandSelection` 등 픽셀 기반 연산이 모두 메인 스레드에서 실행됨.
- **해결**:
  - `TypedArray` 연산을 최적화하고, 불필요한 객체 생성을 줄임.
  - `setInterval` 기반의 애니메이션(Marching Ants)을 `requestAnimationFrame`으로 통합하여 브라우저 주사율에 동기화.

## 4. UI 및 레이어 렌더링 최적화
- **문제**: Layers 패널 등에서 매 렌더링마다 Canvas를 다시 그리는 로직들이 있음.
- **해결**:
  - `React.memo` 또는 `useMemo`를 적극 활용하여 정적인 UI 부분의 재계산 방지.
  - 레이어 썸네일 업데이트 로직을 특정 액션(작업 종료 등) 시에만 실행되도록 변경.

---

### 작업 순서
1. `setMousePos` 리렌더링 문제 해결 (Ref 기반 직접 조작)
2. `drawMarching` 고해상도 처리 부하 해결 (다운샘플링 적용)
3. 애니메이션 루프 최적화 (`setInterval` -> `raf`)
4. 기타 픽셀 연산 루프 최적화 확인
